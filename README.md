# Ed K-P — Link Page

Standalone static site exported from Meridian Link-in-Bio.

## Deploy to GitHub Pages (free)

1. Create a new GitHub repository (e.g. `ekp-links`).
2. Upload **all files in this folder** to the repository root — `index.html`, `styles.css`, the `assets/` folder, and `.nojekyll`.
3. In the repo, go to **Settings → Pages**.
4. Under **Build and deployment**, set **Source** to **Deploy from a branch**.
5. Choose branch **main** (or **master**) and folder **/ (root)**, then save.
6. After a minute or two, your page will be live at:

   `https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/`

### Custom domain (optional)

In **Settings → Pages**, add your domain under **Custom domain** and follow GitHub's DNS instructions.

## What's included

- `index.html` — your link page (fully static, no server needed)
- `styles.css` — compiled styles
- `assets/` — profile photo and link banner images (required for images to show)
- `.nojekyll` — tells GitHub Pages to serve files as-is

## Updating later

Re-export from the Meridian admin when you change your page, then replace the files in this repo and push.
